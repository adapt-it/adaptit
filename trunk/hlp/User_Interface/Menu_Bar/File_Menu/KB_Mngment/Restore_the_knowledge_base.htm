Restore the knowledge base

In the case that an Adapt It knowledge base has been corrupted or contains many obsolete adaptations, Adapt It provides a method to recreate a knowledge base from the documents. Since Adapt It documents contain adaptations that have already been made, all information stored in a project knowledge base is also contained in the project documents.

To use the Adapt It restore tool to recreate a knowledge base from documents do the following:

  1. Start Adapt It normally, until you reach the Document dialog box.

  2. Click Cancel.

    Adapt It opens the project without opening any documents.

  3. If you wish to restore the adaptation knowledge base and not the glossing knowledge base skip to step 6.

  4. To restore the glossing knowledge base, select Allow Glossing from the Advanced menu.

  5. Click the Glossing check box.

  6. In the File menu, select Restore Knowledge Base. (This option is not available if a document is open. Close the document if necessary.)

    A warning message box appears.

  7. Click Yes to proceed (or No to quit the process).

    Adapt It displays the Remove Unwanted Files from the List dialog box. This dialog box allows you to choose the files from which to build the new knowledge base. Remove only the files which might give invalid adaptations. The more files involved; the more complete the knowledge base. For instruction in using the dialog box see Remove Unwanted Files from the List.

    After completion, Adapt It presents another warning message box.

  8. Click Yes to proceed. (Click No if the Yes option causes the application to crash.)

    Adapt It displays a window showing the progress of the restoration process. (If your knowledge base is small you may not see this window.)

    The application then displays a completion dialog box that displays statistics on the restored knowledge base.

  9. Click OK to complete the process.

    You may now use the File menu to open a document.

 Note

  • If your current knowledge base contains obsolete entries that that are not in the documents, restoring the knowledge base rebuilds it with only adaptations or glosses that reside in the documents so obsolete adaptations or glosses are removed.

  • If you have imported knowledge base entries that have not been used yet in any document, you will have to import them again.

  • Another method to restore a knowledge base is to rename the current backup file to become the knowledge base file

  • If Automatic Capitalization is on, Adapt It restores the knowledge base with only lower case entries. All adaptations for both upper case and lower cases forms of a word are contained in the lower case entry.

  • If Automatic Capitalization is off, Adapt It builds a separate entry for the upper case and lower case versions of the word. Thus, the upper case entry of the word contains only the adaptations found with the upper case word and the lower case entry contains only adaptations found with the lower case word.
