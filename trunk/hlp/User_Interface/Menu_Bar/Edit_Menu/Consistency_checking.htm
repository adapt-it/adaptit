Consistency checking

At times it may be necessary to remove or correct entries in the knowledge base. This can occur if incorrect adaptations have been unintentionally entered into the knowledge base. If this occurs the knowledge base is not perfectly consistent with the document contents. This situation does not cause harm in itself; but, the incorrect adaptations remain in the documents. Adapt It provides a method to identify inconsistent translations and then to correct them. The process is to have Adapt It look at all of the adaptations in the documents and find adaptations which do not have a corresponding entry in the knowledge base.

To check the translation consistency, do the following steps:

  1. Open the Edit menu and select Consistency Check…

    Adapt It opens a dialog box and asks if you want to check only the current document or all documents in the project.

  2. Click the option that you want and then click OK to continue.

    Adapt It displays a message asking if you want to save the document you are working on.

  3. Click Yes to save the document.

  4. If you chose the second option (Check this and other documents in the this project),

    Adapt It brings up the Remove Unwanted files from the List dialog box.

    See Remove Unwanted Files from the List for an explanation of how to use this box.

  5. Adapt It checks the adaptations in the document(s) against the knowledge base. When the application finds an inconsistency it opens the Inconsistency Found dialog box. Use the dialog box to make whatever changes are necessary for each inconsistency. See Using the inconsistency found dialog box for information on how to use this dialog box.

 Note

  • Adapt It does not apply any changes you make until all of the inconsistencies are found. Therefore, you may see the same inconsistency several times. This allows you to make a different change each time the inconsistency appears, if you wish.

  • If you wish to have Adapt It automatically make the same change for all instances of the same inconsistency; first, check the Auto-fix later instances the same way check box in the Inconsistency Found dialog. Make the particular change you want and then click OK. Adapt It then automatically makes the changes as necessary and does not bring up the Inconsistency Found dialog.

  • Consistency checking can be tedious if there are many inconsistencies
