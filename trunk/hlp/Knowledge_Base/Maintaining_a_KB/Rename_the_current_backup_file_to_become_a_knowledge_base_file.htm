Rename the current backup file to become a knowledge base file

To use this option you need to have some knowledge of Windows folders and file management. Read the following procedure, before attempting to do it. If you do not understand the procedure seek help from a person with more knowledge.

  1. Locate the Adapt It or Adapt It Unicode Work folder on your computer. This folder is in the My Documents folder.

  2. Locate and open the folder containing the adaptations and knowledge base.

    The name of the project folder has the format: “source language name” to “target language name” adaptations; these names being the names used when you first created the project. For example, French to English adaptations.

  3. Make sure that you see the filename extensions for the files in this folder. If the filename extensions are not visible change the folder options to make them visible.

    See Make filename extensions visible for a procedure to do this in Windows XP. For other operating systems, use the Help menu in the folder to see how it is done.

  4. Locate the knowledge base file. This file has the filename that includes the name of the project and the word adaptations. The filename has a single extension of .XML. Right click the filename and select Delete from the popup menu.

    The computer deletes the file.

  5. Locate the backup knowledge base file. This file has two filename extensions, .BAK followed by .XML. Right Click the filename and select Rename from the popup menu.

  6. Select the characters .BAK (including the period in the selection). Press Delete or Backspace and then click anywhere on the screen to make the change “stick”. Now only .XML should remain as the filename extension.

    Adapt It can recognize this file as the knowledge base. You need to exit the application if you have not already done so. Launch Adapt It again. This time, Adapt It uses the former backup knowledge base as the current knowledge base.

 Warning

  • Changing the filename extension of a file tells windows that the file is of a different type. As a result, Windows may try to use the wrong application with the file with unpredictable results. Do not attempt to change file extension types unless you know that the two file types are compatible.

 Note

  • If this procedure does not help, you can Restore the knowledge base. Restoring the knowledge base quickly recreates a knowledge base using the adaptation documents for the project.
