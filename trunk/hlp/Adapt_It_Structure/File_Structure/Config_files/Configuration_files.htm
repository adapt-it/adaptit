Configuration files

Adapt It requires the use of certain files called configuration files. These files contain data about the projects, documents and your preferences for working with the application. Adapt It builds and maintains these files automatically. You should not attempt to change these files on your own, but only by using the tools provided by Adapt It.

Configuration settings include such things as:

  • Font sizes and colors

  • Preferences about the format of the Main window.

  • Backup and saving preferences.

Adapt It stores the settings you use in two types of configuration files:

  • General Configuration file

  • Project Configuration file

In addition, each document file contains some configuration information. The data contained in these three kinds of files overlap to some degree. The general configuration file contains default preferences for all projects. The project configuration file contains preferences for its project. These preferences override those in the general configuration file. For example, you could change the source font color in a document, and all of the documents in the project would use that color for the source text. The document file contains preferences for the specific document such as the specific filters you wish to use. Adapt It uses the filters for the current document but not for other document. You can set unique filters for any document.

 Note

  • If Adapt It fails on your attempt to launch it, the configuration files may be corrupt. You can rebuild the General Configuration file by pressing the Shift key while launching Adapt It again. Release the Shift key once the Welcome to Adapt It screen appears.

  • To rebuild the Project Configuration file for a specific project, do the same as above but continue to press the Shift key until you reach the Document page of the Start Working wizard. Release the Shift key to rebuild the Project Configuration file.
