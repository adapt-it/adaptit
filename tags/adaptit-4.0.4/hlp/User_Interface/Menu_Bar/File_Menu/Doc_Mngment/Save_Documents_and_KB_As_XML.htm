Save documents and KB as XML

This option is a control to tell Adapt It if documents should be saved in XML format or in binary format. XML is the preferred format, but since earlier version of Adapt It used binary format, there may be some reason to maintain files in that format. Adapt It works equally well with both formats. For more information see XML in Adapt It.

To change the file type of the working documents and knowledge base (KB) do one of the following:

  • If the Save Documents and KB As XML entry in the File menu is checked. Click the item to remove the check mark.

    When Adapt It saves the files, they are saved in binary format.

  • If the Save Documents and KB As XML entry in the File menu is not checked. Click the item to restore the check mark.

    When Adapt It saves the files, they are saved in XML format.

 Note

  • The current version of Adapt It reads either XML or binary files. Adapt It 2.4.1i and earlier can only read binary files.

  • The setting for this command determines only what format is used for saving documents from this point on. It does not change the format of previously saved documents.

  • Adapt It can open either type of file but saves it according to the current setting. Therefore a binary file can be saved as XML or vice versa. However, maintains only one copy of the adaptation file, based on the setting when it was last saved.
